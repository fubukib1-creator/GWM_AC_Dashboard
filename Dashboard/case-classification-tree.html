Case Classification Tree — v3 Dichotomous Key
Visual restatement of design/categorisation-tree.md · 1 root · 8 branches · 23 leaves · classifier in Dashboard/v3/category-engine.js.

Two orthogonal axes every case is classified along both — independently

Axis 1 · this document
Tree path — what kind of case is this?
A dichotomous (multi-way) tree assigns every case to exactly one of 23 leaves based on portal stage and status_customer_id. The classifier is classifyJob() in Dashboard/v3/category-engine.js; the locked spec is design/categorisation-tree.md.
Axis 2 · separate document
SLA status — is it on time or breached?
A 3-way slice (ok / medium / critical) applied to every case independently of which leaf it sits in. Computed by computeSLA(). Visual restatement: sla-requirements.html.
Why two axes, not one. The tree describes case type; SLA describes case timing. A clean delivery (K4.A) can still be a critical SLA breach if any milestone was missed — the leaf and the SLA tag answer different questions and must not be conflated.

The tree 1 root → 4 stages → 4 sub-branches + 5 long-arm leaves → 23 leaves total
